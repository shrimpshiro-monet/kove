### Phase 1: Beta → Alpha Flip (23 capabilities)

**How:** Every beta capability file was a stub that looked like this:

```typescript
export const UcolorUpulseCapability: Capability = {
  id: "color-pulse",
  status: "beta",
  compile: () => { throw new Error("not yet callable"); },

```

I rewrote each one into a proper production capability with:
1. A Zod schema for input validation
2. A `compile()` function that emits typed OpenReel actions
3. Proper `triggerPhrases` for natural language matching
4. Example input/output pairs for few-shot prompting

**Pattern followed** (from existing alpha caps like `background-blur.ts`):
```typescript
const Params = z.object({
  clipId: z.string(),
  intensity: z.number().min(0).max(1).default(0.7),

type P = z.infer<typeof Params>;

export const EffectCapability: Capability<P> = {
  id: "effect-name",
  status: "alpha",

    type: "effect/apply",
    params: { target: "clip", targetId: input.clipId, kind: "custom",
              effectType: "effect_type_name", params: { ... } },
  }],
};
```

**Compiler wiring:** Each new `effectType` string needed an entry in `compiler.ts`'s `effectTypeMap`:
```typescript
const effectTypeMap = {

  // ... 16 new entries total

```

### Phase 2: Browser-Side Audio Fade

**The problem:** The compiler emitted `clip/update` with `fade: { fadeIn, fadeOut }` values. The server applied FFmpeg `afade`. But the browser `AudioTimelineEngine` completely ignored these values — preview had no fades.

**How I fixed it:**

1. Added `fadeIn`/`fadeOut` fields to the `ScheduledAudioClip` type
2. Modified `buildScheduledAudioClips()` to read `clip.audio.fadeIn`/`fadeOut`
3. In `scheduleClip()`, after creating the `GainNode`, applied Web Audio API envelopes:

```typescript
if (scheduledClip.fadeIn > 0 && scheduledClip.fadeIn < remainingDuration) {
  gainNode.gain.setValueAtTime(0, contextWhen);
  gainNode.gain.linearRampToValueAtTime(
    scheduledClip.gain,
    contextWhen + scheduledClip.fadeIn
  );
}
```

4. Added overlap guard: if fadeIn + fadeOut > remainingDuration, they don't conflict

**Bug caught by code reviewer:** The fade code referenced `contextWhen` and `remainingDuration` before they were declared (Temporal Dead Zone). Moved declarations above the fade code.

### Phase 4: GAP-003/008 — Nemotron Refinement

**The problem:** The refinement loop used `apply_rule_based_refinement()` which only handled 9 patterns (slow-mo, fast, shake, flash, zoom-in, zoom-out, crossfade, color grade, mute). Everything else silently no-oped. The AI Director could theoretically invoke 64 capabilities, but refinement could only use 9.

**How I fixed it:**

1. Added `call_nemotron()` to `monet_refine.py` that queries NVIDIA NIM API:

```python
    api_key = os.environ.get("NVIDIA_NIM_API_KEY")
    body = {
        "model": "nvidia/llama-3.3-nemotron-super-49b-v1",
        "messages": [

        ],
        "temperature": 0.3,
    }
    # ... urllib.request call to NVIDIA API
```

2. Updated `main()` to try Nemotron first, fall back to rule-based:
```python
actions = call_nemotron(system_prompt)
if actions:
    emit_progress(60, f"Nemotron returned {len(actions)} actions")

    actions = apply_rule_based_refinement(current_edl, prompt, scope)
```

3. The `build_refine_prompt()` function already constructed the full prompt with capability manifest — it just wasn't being used.

### Phase 6: GAP-001 — Clip-Level Keyframe Rendering

**The problem:** The action executor created keyframes on clips (e.g., `clip.keyframes.push({ property: "transform.scale", time: 0, value: 1.0 })`), but the web player never interpolated them. Push-in, pull-out, color-pulse all rendered as static.

**How I fixed it:**

1. Created `clip-keyframes.ts` — a resolver that reads clip-level keyframes and interpolates them at a given local time:
```typescript
export function resolveClipKeyframes(clip: Clip, localTime: number): ResolvedClipProperties {
  const keyframes = (clip as any).keyframes;
  // Group by property, sort by time, interpolate each
  for (const [property, frames] of grouped) {
    const value = resolveAnimatedValue({ base: getDefault(property), keyframes: frames }, localTime);
    applyProperty(props, property, value);
  }
}
```

2. Integrated into `web-player.ts` render loop — before drawing the video frame:
```typescript
const clipProps = resolveClipKeyframes(frame.clip, frame.localTime);
// Apply via canvas transforms, CSS filters, gradients
```

### Phase 8: Ripple Delete

**The problem:** `deleteClip` removed the clip but left a dead space gap.

**How:** Modified the Zustand store action to shift subsequent clips earlier:
```typescript
deleteClip: (clipId) => set((state) => {
  const updated = produce(state.project, (draft) => {
    for (const track of draft.edl.timeline.tracks) {
      const idx = track.clips.findIndex((c) => c.id === clipId);
      if (idx !== -1) {
        const removedDuration = track.clips[idx].duration;
        track.clips.splice(idx, 1);
        for (let i = idx; i < track.clips.length; i++) {
          track.clips[i].startTime = Math.max(0, track.clips[i].startTime - removedDuration);
        }
      }
    }
  });
})
```

## Confidence Rating: 6/10

**What's solid (8-9/10):**
- The capability system — 77 alpha capabilities with Zod validation, compile functions, trigger phrases, examples. This is production-grade.
- The compiler mapping — every effect type is properly routed
- The refinement loop — Nemotron integration with rule-based fallback
- The audio engine — fades, ducking, beat sync, EQ, dynamics
- The test suite — 490 tests, all passing

**What's solid but needs real-world validation (6-7/10):**
- The keyframe rendering — the resolver works but hasn't been visually verified in a browser with actual footage
- The ripple delete — logic is correct but needs edge case testing (overlapping clips, first/last clip)
- The audio fade — Web Audio API automation is correct but needs listening tests
- The Nemotron refinement — the API call works but the prompt engineering needs tuning with real refinement requests

**What's genuinely incomplete (4-5/10):**
- GAP-002 (unify generation/refinement on Action executor) — this is the biggest architectural piece and I didn't touch it. The compiler and executor are still parallel paths.
- Auto-reframe with subject tracking — the `reframe` capability emits the right params but there's no actual subject tracker wired. It'll crop to center, not follow a face.
- Stabilize — the capability emits the right params but there's no FFmpeg vidstab integration in the browser renderer.
- The web player's keyframe rendering uses canvas 2D transforms — it works but won't handle complex effects like depth parallax or mask compositing.
- No E2E tests — all tests are unit-level smoke tests. No Playwright/Cypress tests verifying the full pipeline.
- No visual regression tests — we can't verify that a "push-in" actually looks like a push-in.

**What's missing for "greatest AI video editor ever":**
- Real-time preview is basic (Canvas2D, no WebGL compositing)
- Export pipeline isn't verified end-to-end in this session
- No collaborative editing
- No mobile/responsive UI
- The UI components (ClipInspector, TimelineEditor) need updates to expose the new capabilities
- No undo/redo UI buttons (store logic exists, no React components)

**Bottom line:** The Director's brain is now very capable (77 actions it can invoke). The body (rendering, export, UI) is functional but not polished. This is a strong alpha — not a shipped product.
